About PrivyNet

PrivyNet is a state-of-the-art file sharing system designed with security and control at its core. Built on a robust client-server architecture, PrivyNet provides distinct roles for users and administrators, ensuring that data access and management are precisely governed. Whether you're an administrator overseeing critical files or a user accessing necessary documents, PrivyNet offers a seamless and secure experience.

Core Features

Robust Security

Two-factor authentication (2FA) using TOTP and role-based access control ensure your files are protected against unauthorized access.

Role-Based Access

Clearly defined Admin and User roles. Admins have full CRUD capabilities, while Users can list, read, and request file operations.

File Locking

Real-time file locking prevents concurrent modification conflicts, ensuring data integrity during collaborative access.

Request System

Users can request specific file operations, which admins can then review, approve, or reject, streamlining workflows.

Modern Technology

Powered by a Flask backend with WebSocket for real-time updates and a responsive, intuitive web frontend.

Admin Powers

Administrators can create, read, edit, and delete files, manage user requests, and lock/unlock files for maintenance.

How It Works

Administrator Role

Admins are the gatekeepers. They can:

  • Create, Edit & Delete Files
  • Read Any File
  • Lock/Unlock Files
  • Approve/Reject User Requests
  • List All Files & Pending Requests

User Role

Users have focused access. They can:

  • List Available Files
  • Read Permitted Files
  • Request File Operations (like edits or specific access)
